# Nexus v2.1 — Language-Agnostic Invisible Infrastructure

> Approved 2026-04-12. Evolves nexus from a Node-centric project tool into language-agnostic invisible infrastructure with zero repo footprint, smart detection, and living context for AI agents.

---

## Problem

Nexus v2 works but has three problems:

1. **Repo pollution.** Init drops 7 scripts into `scripts/`, a justfile with placeholder commands, and config files. A developer's first impression is "what is all this stuff" not "this is helping me."

2. **Node-only.** Four of seven scripts require Bun. The orphaned files checker has hardcoded Next.js filenames. Doctor recommends eslint/prettier even for a Go project. A Python developer running `nexus init` gets TypeScript files they can't use.

3. **Static context.** CLAUDE.md is conventions written at init time. A fresh AI agent landing in the project doesn't know the current state — what stack is being used, how many files exist, what the last health check found. The agent has to discover everything from scratch every session.

## Solution

Three layered changes, each valuable independently:

1. **Zero-footprint.** Scripts live in `~/.nexus/scripts/` and never enter the user's repo. Hooks reference global paths. The user sees CLAUDE.md, hook configs, and that's it.

2. **Smart detection.** Nexus detects the project ecosystem from files on disk and only runs checks that apply. No Node checks in a Python project. No placeholder content. Silence when there's nothing useful to say.

3. **Living context.** A `.nexus/context.md` file (gitignored) that syncs at session start, session stop, and pre-commit. Fresh AI agents read it and immediately know the project state — stack, structure, health — without anyone writing documentation.

---

## Principle

Nexus is not a scaffolder. It doesn't tell you how to code, what framework to use, or what your conventions should be. It's the invisible web that keeps a project self-maintaining from the moment it starts. Superpowers does the thinking. Nexus does the plumbing.

## Change 1: Zero-Footprint

### Scripts stay in `~/.nexus/scripts/`

Scripts are never copied into the user's project. All hooks reference them via `~/.nexus/scripts/`:

```yaml
# lefthook.yml
pre-commit:
  parallel: true
  commands:
    env-sync:
      run: bash ~/.nexus/scripts/check-env-sync.sh
    deps-direction:
      run: "[ -f package.json ] && bun ~/.nexus/scripts/check-deps-direction.ts || true"
```

```json
// .claude/settings.json (full version — without Superpowers)
{
  "hooks": {
    "PreToolUse": [
      {
        "matcher": "Bash|Read|Glob|Grep",
        "command": "bash ~/.nexus/scripts/session-sync.sh 2>/dev/null || true"
      }
    ],
    "onStop": [
      {
        "command": "nexus doctor --quick 2>/dev/null || true"
      }
    ]
  }
}
```

Note: The `PreToolUse` hook fires on the first matching tool use each session. The `session-sync.sh` script internally tracks whether it's already run (via a timestamp file in `.nexus/`) to avoid redundant syncs within the same session.

If nexus isn't installed, hooks fail silently (`|| true`). The project still works — it just doesn't have enforcement.

### Superpowers detection

Same as v2: if `~/.claude/plugins/` contains superpowers, strip the `onStop` hook from `.claude/settings.json` (Superpowers handles session-end behavior itself). The `PreToolUse` session sync hook stays regardless — Superpowers doesn't manage context sync.

### What nexus init drops into the project

| File | Purpose |
|------|---------|
| `CLAUDE.md` | Conventions the AI agent follows |
| `.claude/settings.json` | Claude Code hooks (session sync, doctor on stop) |
| `lefthook.yml` | Pre-commit hooks |
| `.gitignore` | Standard ignores + `.nexus/` |
| `.env.example` | Env var documentation |
| `.mise.toml` | Pinned runtimes |
| `.github/pull_request_template.md` | PR review checklist |

### What it stops dropping

- `scripts/` directory (7 files) — gone from user's repo
- `justfile` — only dropped if no existing build system detected. Existing build systems: `Makefile`, `taskfile.yml`, `justfile`, `Rakefile`, `build.gradle`, `pom.xml`. If any exist, skip. When dropped, contains only `nexus doctor` and `nexus doctor --fix` commands — no placeholder dev/build/test commands.

### Checksum system

The `.nexus-checksums` file for tracking customized scripts is removed. There's nothing to customize in the project — scripts live globally. `nexus update` for scripts becomes `git pull` on `~/.nexus/` and all projects benefit immediately.

### `.nexus-version`

Still stamped in the project to track which version of nexus initialized it. Used by `nexus update` for migration decisions.

## Change 2: Smart Detection

### Ecosystem detection

Nexus looks at files on disk to determine the project ecosystem:

| File | Ecosystem | ID |
|------|-----------|-----|
| `package.json` | Node/Bun | `node` |
| `go.mod` | Go | `go` |
| `pyproject.toml` or `requirements.txt` | Python | `python` |
| `Cargo.toml` | Rust | `rust` |
| None of the above | Generic | `generic` |

Detection runs at init, at session start (context sync), and during doctor. The result is stored in `.nexus/context.md`.

### What adapts per ecosystem

**Doctor checks:**

| Check | node | go | python | rust | generic |
|-------|------|-----|--------|------|---------|
| CLAUDE.md file structure sync | yes | yes | yes | yes | yes |
| .env.example coverage | yes (process.env, Bun.env) | yes (os.Getenv) | yes (os.environ) | yes (std::env) | skip |
| Dependency direction | yes | skip | skip | skip | skip |
| Hallucinated imports | yes | skip | skip | skip | skip |
| Dead exports | yes | skip | skip | skip | skip |
| Orphaned files | yes | skip | skip | skip | skip |
| Nexus version | yes | yes | yes | yes | yes |

Checks that don't apply don't appear in the output at all — no "skip" lines for irrelevant checks.

**Doctor recommendations:**

Universal (all ecosystems):
- No CLAUDE.md → `Run: nexus init`
- No pre-commit hooks → `Run: nexus init`
- No .env.example → `Run: nexus doctor --fix`

Node only:
- No linter → `pnpm add -D eslint`
- TypeScript strict mode off
- No formatter → `pnpm add -D prettier`
- No test runner → `pnpm add -D vitest`
- No engines field in package.json
- No `"type": "module"` in package.json

Recommendations for other ecosystems can be added later. The framework is there.

**CLAUDE.md conventions:**

Universal conventions (all ecosystems):
```markdown
<!-- nexus:conventions -->
## Conventions
- No business logic in route handlers, API handlers, or UI components.
- No hardcoded secrets or credentials.
<!-- nexus:end -->
```

Node-specific conventions (appended only when Node detected):
```markdown
<!-- nexus:conventions-node -->
## Node Conventions
- All API responses follow: `{ data, error, metadata }`
- All errors follow: `{ code, message, details }`
- All database tables have: `id`, `created_at`, `updated_at`, `deleted_at`
- No `any` types. No `@ts-ignore`. No `eslint-disable` without a comment explaining why.
- No direct database calls outside repositories.
<!-- nexus:end -->
```

Generic projects get just the universal conventions. No empty sections, no placeholder content.

**Lefthook hooks:**

Universal:
```yaml
pre-commit:
  parallel: true
  commands:
    env-sync:
      run: bash ~/.nexus/scripts/check-env-sync.sh
    claude-md-sync:
      run: bash ~/.nexus/scripts/sync-claude-md.sh
```

Node (appended when detected):
```yaml
    deps-direction:
      run: "[ -f package.json ] && bun ~/.nexus/scripts/check-deps-direction.ts || true"
    hallucinated-imports:
      run: "[ -f package.json ] && bun ~/.nexus/scripts/check-hallucinated-imports.ts || true"
    typecheck:
      run: "[ -f package.json ] && pnpm typecheck || true"
    lint:
      run: "[ -f package.json ] && pnpm lint || true"
```

### check-env-sync.sh adaptation

The env sync script needs to detect which pattern to grep for:

```bash
# Detect env var patterns based on ecosystem
if [ -f "package.json" ]; then
  PATTERN='(process\.env|Bun\.env)\.[A-Z_][A-Z0-9_]*'
elif [ -f "go.mod" ]; then
  PATTERN='os\.Getenv\("[A-Z_][A-Z0-9_]*"\)'
elif [ -f "pyproject.toml" ] || [ -f "requirements.txt" ]; then
  PATTERN='os\.environ\[?"[A-Z_][A-Z0-9_]*"'
elif [ -f "Cargo.toml" ]; then
  PATTERN='std::env::var\("[A-Z_][A-Z0-9_]*"\)'
else
  exit 0  # No ecosystem detected, skip
fi
```

### check-orphaned-files.ts adaptation

The hardcoded Next.js convention files list needs to be framework-aware:

- Detect Next.js (`next.config.*` exists) → use current Next.js convention list
- Detect Remix (`remix.config.*` or `app/root.tsx`) → use Remix conventions
- Detect SvelteKit (`svelte.config.*`) → use SvelteKit conventions
- No framework detected → skip convention file exclusion, only exclude index/config files

## Change 3: Living Context

### The file: `.nexus/context.md`

Auto-generated, never hand-edited. Gitignored.

```markdown
# Project Context
<!-- Auto-maintained by nexus. Do not edit. -->

## Stack
- Runtime: Node 24 (from .mise.toml)
- Framework: Next.js 15.2 (from package.json)
- Database: Drizzle + Postgres (from drizzle.config.ts)
- Auth: Clerk (from @clerk/nextjs in package.json)

## Structure
- 47 source files across 12 directories
- Entry points: src/app/layout.tsx, src/app/page.tsx
- Last sync: 2026-04-12T14:22:00

## Health
- Last doctor: 2026-04-12T14:22:00
- Result: 7/7 passed
- Recommendations: No test runner, TypeScript strict off
```

### Stack detection

For Node projects, parse `package.json` dependencies:

| Dependency | Detected as |
|-----------|-------------|
| `next` | Framework: Next.js {version} |
| `express` | Framework: Express {version} |
| `hono` | Framework: Hono {version} |
| `drizzle-orm` | Database: Drizzle |
| `prisma` | Database: Prisma |
| `@clerk/nextjs` | Auth: Clerk |
| `next-auth` | Auth: NextAuth |
| `lucia` | Auth: Lucia |

For other ecosystems, detect from config files and manifests similarly.

### Sync triggers

| When | What runs | Script |
|------|-----------|--------|
| Session start | Full context sync + CLAUDE.md file structure | `session-sync.sh` (via PreToolUse `runOnce`) |
| Session stop | Doctor quick + context health update | `nexus doctor --quick` (via onStop) |
| Pre-commit | CLAUDE.md file structure sync | `sync-claude-md.sh` (via lefthook) |

**No PostToolUse sync.** The file structure doesn't need to be real-time. The agent knows what it wrote in the current session. Syncing on every write wastes cycles for no benefit. Context is fresh at the boundaries that matter: session start, session stop, and commit.

### session-sync.sh (new script)

Runs once at the start of each Claude Code session:

1. Detect ecosystem (check for package.json, go.mod, etc.)
2. Detect stack (parse dependencies, find framework/db/auth)
3. Count source files and directories
4. Read last doctor result (if exists)
5. Write `.nexus/context.md`
6. Run `sync-claude-md.sh` to update file structure

### CLAUDE.md pointer

CLAUDE.md gets a new nexus-owned section:

```markdown
<!-- nexus:context -->
## Project Context
See `.nexus/context.md` for auto-detected project state (stack, structure, health).
Updated at the start of each session and after each doctor run.
<!-- nexus:end -->
```

### New files
- `scripts/session-sync.sh` — session start context sync
- `templates/CLAUDE.md.node` — Node-specific convention additions
- `templates/lefthook.yml.node` — Node-specific hook additions

### Modified files
- `nexus` — no changes
- `cli/init.sh` — stop dropping scripts, add detection, add git remote hint, adapt output per ecosystem
- `cli/doctor.sh` — detection-aware checks, remove skip lines for irrelevant checks, adapt recommendations
- `cli/update.sh` — remove script checksum migration (scripts are global now), keep CLAUDE.md section migration
- `cli/helpers.sh` — add `detect_ecosystem()` function, remove `drop_file_silent` (not needed, fewer files to drop), remove checksum functions

- `scripts/sync-claude-md.sh` — no changes

## Out of Scope

- Ecosystem-specific doctor recommendations beyond Node (add in future versions)
- Equivalent import/export checks for Go/Python/Rust (add as scripts mature)
- Scaffolding or project templates (not what nexus does)
- Controlling Superpowers behavior (nexus provides context, Superpowers decides what to do)
- GUI or TUI
